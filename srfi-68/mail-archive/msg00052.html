My personal idea about this for I/O is that it might be most useful if the new
stuff solves the problem as it should be solved---but preferably in a way
that the old stuff (R5RS I/O) can be implemented on top of the new stuff
for backwards compatibility.

I don't oversee all the details right now, but that should be doable here.
