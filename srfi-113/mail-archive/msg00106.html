I have pushed new versions of SRFI 113 and its implementation to
<http://www.ccil.org/~cowan/temp/srfi-113.html> and
<http://www.ccil.org/~cowan/temp/sets.tar.gz>.

The only change to the SRFI is a note explaining that the implementation
now registers its comparators with the default comparator of SRFI 114.
This makes the implementation dependent in a minor way on the sample
implementation of SRFI 114, but removing two lines of code from
sets-impl.scm will remove this dependency.  If that is done, alternative
means should be used to make sets and bags available to the default comparator.

Tests have been provided for the comparators.
