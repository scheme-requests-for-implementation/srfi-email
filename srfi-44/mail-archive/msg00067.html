A new draft-draft is available at http://sgmiller.org/srfi-44.html.  
This fixes a few typos, and changes collection-fold-keys-* to require 
that the fold function accept both the key and value simultaneously 
rather than requiring a programmer to retrieve the value using dict-get 
or sequence-ref.  

I feel we are very close to a final SRFI.  If there are no further 
comments, I think we can close the SRFI to non-typo fixes, and move to 
finalize once the reference implementation is brought inline with the 
spec and its sufficiently tested.
